Problem Setup

The Environment

Before a robot can move, it needs to understand its environment. We define the start point, the goal point, and the boundaries.

In autonomous navigation, a robot is given a task: to move from a starting point to a target destination.

However, the environment is not always empty. There may be obstacles such as walls, objects, or restricted areas that the robot cannot pass through.

This creates a navigation problem: how can the robot reach its goal safely and efficiently without colliding with obstacles?

To solve this problem, the real world is simplified into a digital environment that the robot can understand, such as a map with clear boundaries and constraints.

In the next sections, we will see how this environment is represented using a grid map and how algorithms help the robot find the best path.

A simplified abstraction of a robot navigation environment, showing the start state, goal state, obstacles and boundaries.

Grid Map Representation

Digital representation of the environment for robot navigation

In robot navigation, the real world must first be converted into a digital representation that a computer can understand. One of the most common and simple representations is the grid map.

A grid map divides the environment into equal-sized cells. Each cell stores information about whether the robot can move into that area or if it is blocked by an obstacle.

Digital Representation (Real World ‚Üí Data)

Robots cannot reason directly about physical space. Instead, sensor data is converted into numerical values that describe the environment. In a grid map, each cell is represented digitally using simple values.

  • 0 ‚Äì free space
  • 1 ‚Äì obstacle
  • S ‚Äì start position of the robot
  • G ‚Äì goal or destination
Key idea: Once the environment is represented as data, path planning algorithms such as A* can compute a path logically and efficiently.
Cell Meanings in a Grid Map
  • Free cells allow robot movement
  • Obstacle cells represent walls or blocked areas
  • Start cell (S) marks where the robot begins
  • Goal cell (G) marks the target destination

A grid map example where S is the start position, G is the goal, and blue coloured cells represent obstacles.

Why are grid maps important?

Grid maps simplify navigation by breaking continuous space into discrete steps. This makes robot movement easier to analyse and enables efficient path planning.

A* Algorithm Concept

Optimal path planning using cost + heuristic

A* (A-star) is a pathfinding algorithm that finds the shortest path from a start node to a goal node. It is widely used in robotics, AI and navigation because it balances explored cost and estimated cost.

Cost Function

A* evaluates nodes using:

f(n) = g(n) + h(n)
  • g(n): actual cost from Start ‚Üí current node
  • h(n): heuristic estimate from current node ‚Üí Goal
  • f(n): total estimated cost via that node (A* picks the lowest f(n) next)
How A* works (high-level)
  1. Start from the Start cell and place it into an ‚Äúopen list‚Äù.
  2. Pick the node with the lowest f(n) from the open list.
  3. Expand its neighbours and update their costs if a better path is found.
  4. Move expanded nodes to a ‚Äúclosed list‚Äù and repeat until the Goal is reached.

Nodes start in the Open List (discovered) and move to the Closed List once they are expanded.

Tip: If the heuristic h(n) never overestimates the remaining distance (admissible), A* is guaranteed to return an optimal shortest path.
