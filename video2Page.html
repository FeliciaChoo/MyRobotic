A* Algorithm Concept

Optimal path planning using cost + heuristic

A* (A-star) is a pathfinding algorithm that finds the shortest path from a start node to a goal node. It is widely used in robotics, AI and navigation because it balances explored cost and estimated cost.

Cost Function

A* evaluates nodes using:

f(n) = g(n) + h(n)
  • g(n): actual cost from Start → current node
  • h(n): heuristic estimate from current node → Goal
  • f(n): total estimated cost via that node (A* picks the lowest f(n) next)
How A* works
  1. Start from the Start cell and place it into an “open list”.
  2. Pick the node with the lowest f(n) from the open list.
  3. Expand its neighbours and update their costs if a better path is found.
  4. Move expanded nodes to a “closed list” and repeat until the Goal is reached.
Tip: If the heuristic h(n) does not overestimate the remaining distance, A* will always find the shortest possible path.

How the Path Length is Calculated

A* does not directly calculate the final path length using the formula. Instead, it uses the cost function below to decide which grid cell to explore next.

f(n) = g(n) + h(n)
  • g(n) – number of steps from Start to the current cell
  • h(n) – estimated remaining distance to the Goal
  • f(n) – score used to prioritise which cell to explore

Once the Goal is reached, the algorithm stops and the final path length is taken from g(goal).

In this simulation: The robot reached the goal with g(goal) = 18, meaning the shortest path contains 18 steps.
Grid Map Representation

The environment is represented as a two-dimensional grid map, where each square corresponds to a possible position the robot can occupy.

  • Each grid cell represents a discrete location in the environment.
  • The robot can move between neighbouring cells in four directions: up, down, left, and right.
  • Some cells are blocked by obstacles and cannot be traversed.

Cost Function

The Math: f(n) = g(n) + h(n)

How does the robot decide which step to take? It calculates the cost:

  • g(n): Cost from start to current node.
  • h(n): Heuristic estimated cost to the goal.
